# Upstream 0.45 Fork Integration Design

**Status:** Completed on 2026-07-19. The validated fork head is
`bfaaa1bbc85e47da020f435b9b5f9e319d529f2d`; the upstream base and safety branch remain pinned below.

## Goal

Move Nico's CodexBar fork from 0.41.0/build 100 to the pinned upstream 0.45.0/build 107 commit
`2ccb4525687c92ff1cd50c8c57f24420c1fcb71f` while preserving only fork behavior that is still needed.

## Integration strategy

Build on a clean upstream branch instead of merging the old fork history. The pre-integration state is preserved and
pushed as `codex/pre-upstream-0.45-safety-20260718` at `9bbef21c877122f3e291128928781e3f3104eff0`.
Port each surviving customization as a focused change with focused tests. Do not reintroduce code removed upstream,
including the Kimi K2 relay, or fixes that upstream 0.45 already supersedes.

## Surviving fork behavior

- Keep Codex cards compact by hiding Codex extra quota rows, Codex credit balances, and buy-credit actions.
- Hide the noisy Claude routines row while retaining other Claude model-specific rows.
- Bucket status-item observation signatures at the same granularity as rendered icon cache keys.
- Present an exhausted Claude extra-usage spend cap as the primary blocking window.
- Keep passive Keychain reads non-interactive and treat `errSecInteractionNotAllowed` as unavailable data.
- Remove deprecated trusted-application ACL attachment from CodexBar cache writes.
- Avoid rewriting an unchanged cookie cache entry solely to advance `storedAt`.
- Prefer a valid installed signing identity when a stale or unusable `APP_IDENTITY` is configured.
- Keep fork-only GitHub automation and bundled agent release skills absent.

## Upstream behavior retained

- Keep upstream 0.45 provider additions, menu-layout editor, agent sessions, spend dashboard, adaptive refresh, and
  quota forecasting unchanged.
- Keep upstream cookie refresh transaction and validation semantics.
- Keep upstream Kimi K2 and CrossModel removals.
- Keep upstream cancellation handling and current Claude OAuth/keychain architecture where it supersedes old fork
  patches.

## Test isolation fix

`ClaudeOAuthDelegatedRefreshRecoveryTests` must apply a task-local non-never prompt mode before seeding or reading
the CodexBar OAuth cache. The tests must not depend on the machine's stored Claude prompt preference and must never
fall through to a real Keychain read.

## Validation and rollout

Run focused tests after each behavior port, then `make check` and
`./Scripts/compile_and_run.sh --test --wait`. Validate the repo-local 0.45.0/build 107 bundle before replacing
`/Applications/CodexBar.app`. After installation, verify the running executable path, version/build, code signature,
login-item registration, and a quiet refresh interval without Keychain prompt events. Push the validated integration
branch before advancing `fork/main`.
