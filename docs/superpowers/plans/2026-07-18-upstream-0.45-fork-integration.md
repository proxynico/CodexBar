# Upstream 0.45 Fork Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a validated Nico fork on pinned upstream 0.45.0/build 107 while retaining only approved fork behavior.

**Architecture:** Use `2ccb4525687c92ff1cd50c8c57f24420c1fcb71f` as the clean base. Port each
surviving customization through its stable model or storage seam, preserve upstream 0.45 architecture, and validate
the final repo-local bundle before installing it.

**Tech Stack:** Swift 6, Swift Testing/XCTest, SwiftPM, AppKit/SwiftUI, Security.framework, bash, Xcode widget build.

## Global Constraints

- Do not reintroduce Kimi K2 or CrossModel relay code removed upstream.
- Do not run live provider probes or tests that can show macOS Keychain prompts.
- Keep all Security.framework cache reads non-interactive.
- Do not introduce dependencies or unrelated refactors.
- Rebuild and restart with `./Scripts/compile_and_run.sh --wait` after every edit batch.
- Run `make check` and `./Scripts/compile_and_run.sh --test --wait` before installation.
